# email-utils 컨벤션 (디폴트 — develop 스킬이 생성)

> 신규 프로젝트라 기존 코드 패턴이 없어 디폴트로 생성됨. 팀 결정에 따라 수정할 것.

## 구조
- `src/` — 구현 (모듈당 파일 하나, kebab-case 파일명)
- `test/` — 테스트 (`<모듈명>.test.js`, node:test 사용)

## 코드 스타일
- 순수 Node.js (외부 의존성 최소화), ESM(`import/export`)
- 함수는 camelCase, 한 함수는 한 가지 일만
- 입력 검증 실패 시 throw 대신 안전한 기본 동작을 우선 (각 함수 JSDoc에 명시)

## 테스트
- 실행: `npm test` (= `node --test test/`)
- 정상 케이스 + 경계/비정상 케이스를 반드시 포함

## 커밋/PR
- 커밋 메시지 한글, 브랜치는 `feat/<티켓키>-<요약>` 형식
- PR 본문: 요약 / 변경사항 / 테스트 결과 / 티켓 링크
