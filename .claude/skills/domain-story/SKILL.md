# Domain Story (사람이 읽는 도메인 지식 문서)

`domain` 스킬이 만드는 **코딩 레디 레퍼런스**(개발자/AI가 소스 없이 구현하는 문서)와 달리,
이 스킬은 **비개발자(기획자·신규 입사자)가 읽히고, 나중에 RAG로 질문하면 답이 되는 서사형
도메인 지식 문서**를 만든다. 같은 코드를 보지만 산출물의 독자와 형태가 다르다:
코드 식별자가 아니라 **업무 언어로 번역**하고, 필드 나열이 아니라 **이야기·용어집·시나리오·FAQ**로 쓴다.

핵심 원칙은 `domain`과 같다 — **코드에서 확인된 사실만 문서가 된다**. 코드로 알 수 없는
비즈니스 "왜"는 지어내지 않고 `[확인 필요]`로 남긴다(사용자 인터뷰는 하지 않는다).
모든 문서는 `references/story-template.md`의 **서사형 골격 + RAG 작성 규칙**을 따른다.

문서 위치는 `domain`과 동일하게 **대상 프로젝트 repo**다 — 도메인당 파일 하나
(`docs/story/<영문-슬러그>.md`, 큰 도메인은 `docs/story/<슬러그>/`로 분할). `docs/domain/`(개발자용)과
분리해, 추후 Confluence 발행 스킬이 `docs/story/`를 위키 소스로 export하기 쉽게 한다.

참조: domain-split(분해 방법·측정 루브릭) / story-template(골격·RAG) / story-craft(번역·서사 작법) /
example-story(본보기) / `../domain/references/extraction-discipline.md`(관련성·멀티소스·SQL은닉) /
`../domain/references/method-ddd.md`(BC 경계 신호 — 분해 1차 근거).

## 인자 / 모드 판별

`$ARGUMENTS`를 확인한다:

- 토큰 중 **실재하는 디렉터리 경로**가 있으면 → 대상 프로젝트 경로(1순위·필수). 없으면 사용자에게
  물어본다 (이 OS 저장소 자신은 대상이 아니다 — 문서는 대상 프로젝트 repo에 산다)
- 나머지 토큰이 **도메인 이름**이다. 주어지면 분해 단계를 건너뛰고 그 도메인만 다룬다(여러 개면 각각)
- **도메인 이름이 없으면** → 1단계(도메인 자동 분해)부터 수행한다
- **모드는 기존 문서 존재 여부로 자동 판단한다**: `docs/story/<슬러그>.md`(또는 디렉터리)가
  - 없으면 → **bootstrap**(신규), 있으면 → **update**(기존을 베이스로 갱신)
  - 인자가 `rebuild`로 시작하면 → 있어도 **bootstrap 전면 재구축**(기존은 참고 자료)
- **테스트 모드**: 사용자가 "테스트" 또는 `docs/temp/`를 지정하면 산출물을 `docs/temp/<슬러그>.md`로
  저장한다 (실서비스 경로 `docs/story/`를 더럽히지 않고 품질을 확인하기 위함)

## 절차

1. **도메인 자동 분해 (domain-splitter 서브 에이전트) — 도메인 이름이 없을 때만**
   - **domain-splitter**(Agent 도구, subagent_type: `domain-splitter`)에 대상 경로와
     `references/domain-split.md` 경로를 전달해 **하이브리드(BC 1차 + 주제 2차) 후보 분해안**을 받는다.
     시야를 넓히려면 splitter를 **복수 호출**해도 된다(예: DDD-경계 시야 / 모듈-구조 시야)
   - **메인이 직접 채점·선택한다**(사람에게 묻지 않음): `domain-split.md`의 측정 루브릭
     (응집도·결합도·용어경계·페이지 적정크기·질문 커버리지·전체 균형·고아코드 없음)으로 각 후보를
     채점하고 **게이트=최저 차원**으로 최적안을 고른다
   - **선택 후 한 번 더 검증**: 어느 도메인에도 안 들어간 모듈(고아), 두 도메인에 걸친 모듈(경계 누수),
     1~2개로 뭉치거나 과도하게 흩어진 과대/과소 분해가 없는지 점검. 문제가 있으면 splitter를 다시 돌린다
   - 확정된 도메인 목록(슬러그·제목·포함 경로·별칭·한 줄 책임)을 사용자에게 **통보**하고 다음으로 진행

2. **도메인별 초안 작성 (domain-storyteller 서브 에이전트)** — 확정된 도메인마다
   **domain-storyteller**(subagent_type: `domain-storyteller`)를 spawn한다(병렬 가능). 전달 항목:
   - 모드(bootstrap|update), 대상 프로젝트 경로, **단일 도메인과 그 포함 경로/모듈**
   - 레퍼런스 경로: `story-template.md` + `story-craft.md` + `example-story.md`
     + `../domain/references/extraction-discipline.md`(항상)
   - update면: 기존 문서 경로 + 기준 커밋(`git log -1 --format=%H -- docs/story/<슬러그>*`)
   - 분석가는 서사형 핸드북 초안(또는 update면 섹션별 변경 제안)을 반환한다

3. **초안 자체 점검** — 반환된 초안을 점검한다:
   - 본문 사실에 근거 표기 `(근거: 파일:라인)`가 빠졌으면 "미확인·열린 질문"으로 강등
   - 코드 식별자가 그대로 노출된 문장(번역 안 됨)·정의 없는 전문용어가 있으면 표시
   - **가독성 루브릭**(story-template) 점검: 코드명 몰라도 이해되나·전문용어 인라인 정의·규칙마다
     숫자 예시·시나리오가 서사·FAQ가 실제 질문 예측·각 섹션 self-contained·마크다운 유효(표 헤더=셀)

4. **사람 검수 게이트 (내용 한정 — 저장 전 필수)** — 도메인별 순차 진행:
   - 초안 전문을 사용자에게 보여준다
   - **`[추정]`/`[확인 필요]`/`[제안]`을 AskUserQuestion으로 전수 처리**(≤4개씩):
     "사실 맞음(확정) / 수정 필요(직접 입력) / 틀림(삭제) / 나도 모름([확인 필요]로 남김)"
   - ※ **도메인 분해는 검수 대상이 아니다**(메인이 이미 측정·검증함). 비즈니스 "왜" 인터뷰도 하지 않는다
   - 처리 후 AskUserQuestion으로 **"이대로 저장할까요?"** 최종 확인

5. **저장** — 검수 통과본을 `docs/story/<영문-슬러그>.md`(테스트면 `docs/temp/<슬러그>.md`)로 저장한다
   (디렉터리 없으면 생성). story-template의 frontmatter(domain·title·aliases·audience·source_repo·
   base_commit·updated·related_docs)를 채운다. 분량이 크면 `docs/story/<슬러그>/`로 분할 + README 허브.
   기준 커밋은 현재 HEAD(`git -C <대상경로> rev-parse --short HEAD`), 갱신일은 오늘

6. **커밋 제안** — AskUserQuestion으로 커밋 여부를 물어본다(테스트면 생략 가능):
   - **예** → 대상 프로젝트에서 `docs/story-<도메인>` 브랜치를 만들어 한글 메시지로 커밋(commit 스킬 스타일).
     이어서 push·PR 여부를 물어본다
   - **아니오** → 파일 저장까지만

7. **완료 보고(한글)** — **채택된 분해안과 점수·탈락안 사유**(분해를 수행했으면) + 도메인별 문서 경로·모드·
   검수에서 확정/수정/삭제된 추측 개수·남은 `[확인 필요]` 수. `[확인 필요]`가 남으면 "도메인 전문가에게
   물어볼 것" 목록으로 정리한다

## 주의사항

- **내용 검수 없이 저장하지 않는다** — 4단계(사람 검수 게이트)를 건너뛰고 파일을 쓰지 않는다
- **도메인 분해는 사람에게 묻지 않고** 메인이 측정·검증해 진행하되, **측정 근거(점수표·탈락안)를 보고에 남긴다**
- 대상 프로젝트의 main/master에 직접 커밋·push하지 않는다 — 반드시 브랜치
- **문서 외의 파일은 건드리지 않는다** — 발견한 코드 문제는 보고만
- **코드로 알 수 없는 "왜"는 지어내지 않는다** — `[확인 필요]`로 남긴다(매끈한 거짓 < 구멍 보이는 진실)
- **코드 식별자를 그대로 쓰지 않는다** — 업무 언어로 번역하고, 코드명은 참고용으로만 괄호에 (story-craft)
- 보고와 문서는 한글로 작성한다 (코드 식별자·고유명사는 원문 유지)
- Confluence/Atlassian 발행은 이 스킬의 범위 밖이다 — 로컬 마크다운까지만 만든다
